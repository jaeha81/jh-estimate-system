# 브랜드 샘플 분석 기록 (brand_sample_log.md)

> **작성 방법:** 재하가 직접 브랜드별 엑셀 견적서를 열어 아래 양식에 맞게 분석 기록
> 목표: 4~5개 브랜드 완료 → Wave 3에서 brand_profiles DB 입력

---

## 분석 체크리스트 (브랜드별 필수 확인 항목)

- [ ] 시트 목록 (탭 이름 전부)
- [ ] 세부내역 시트 이름
- [ ] 세부내역 시트의 데이터 시작 행 번호
- [ ] 각 열 위치 (품명/규격/단위/수량/단가/금액/재료비/노무비/경비)
- [ ] 행 수 고정 여부
- [ ] 병합 셀 존재 여부
- [ ] 수식 셀 존재 여부

---

## 브랜드 #1 — [브랜드명 또는 클라이언트명 입력]

**파일명:** `파일명.xlsx`
**수집일:** 2026-XX-XX
**이미 값 채워진 샘플 여부:** ✅ / ❌

### 시트 목록
| 순서 | 시트 이름 | 역할 추정 |
|---|---|---|
| 1 | Sheet1 | 갑지 |
| 2 | 세부내역서 | 세부내역 ← 핵심 |
| 3 | ... | ... |

### 세부내역 시트 상세
- **시트명:** `세부내역서`
- **데이터 시작 행:** 행 번호 (예: 7번째 행부터)
- **총 데이터 행 수 (대략):** XX행

### 열 위치 매핑
| 필드 | 열 | 비고 |
|---|---|---|
| 품명 (item_name) | B | |
| 규격 (spec) | C | |
| 단위 (unit) | D | |
| 수량 (qty) | E | |
| 재료비 단가 | F | 없으면 — |
| 노무비 단가 | G | 없으면 — |
| 경비 단가 | H | 없으면 — |
| 단가 (unit_price) | I | |
| 금액 (amount) | J | |

### 구조 특이사항
- 행 수 고정 여부: ✅ 고정 / ❌ 가변
- 병합 셀 존재: ✅ 있음 / ❌ 없음
- 수식 셀 존재: ✅ 있음 (J열 금액 자동계산) / ❌ 없음
- 기타: (특이사항 자유 기록)

### brand_profiles JSONB 예시 (작성 완료 후 채움)
```json
{
  "brand_name": "브랜드명",
  "sheet_mapping": {
    "세부내역": "세부내역서"
  },
  "column_mapping": {
    "item_name": "B",
    "spec": "C",
    "unit": "D",
    "qty": "E",
    "unit_price": "I",
    "amount": "J"
  },
  "fixed_row_count": true
}
```
